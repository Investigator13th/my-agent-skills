# 元技能：如何为本仓库贡献 Skills (Meta-Skill)

**系统提示**：如果你是一个正在为本仓库添砖加瓦的 AI Agent，请在进行任何写入操作时严格遵守此规范！

## 1. 结构与归类策略
- 深入理解用户的意图，将新产生的技能归类到现有的业务场景极点中，如 `research/`, `live-ecommerce/`, `fund/`, `daily-life/`。
- 如果不属于现有类别且具备重要独立性，请提议新建顶层目录，内置 `README.md`。

## 2. 内容呈现规范
对于所有的 `.md` 提示词或技能组件：
- 必须包含**大标题**。
- 必须包含**场景说明**和**关键词**（强化检索效果）。
- 如果是提示词，使用 `> ` 引用格式标出主要 Prompt，并以 `{{这里}}` 标出需要用户填写的占位符。
- 命名规则示例：`03_literature_review.md`（采用数字编号_英文命名）。

## 3. 动态维护索引
- 在添加新技能或新目录后，你**必须主动修改**仓库根目录的 `README.md` 和 `index.md`，将新的文件路径建立链接补充到【索引列表】中。

## 4. Git 提交规范
- 请将修改内容打包 commit： `feat(分类): 增加 [技能名称] 技能支持`。
