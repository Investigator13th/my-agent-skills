# 单页应用快速构建工作流

本工作流适用于 **快速原型开发、静态页面演示、简单的单页工具** 等场景。

## 流程概览

1. **需求分析** (PRD)
2. **UI 设计** (Design)
3. **HTML 实现** (Implementation)

---

## 阶段 1：需求明确

### 1.1 确认核心功能
无需编写完整的 PRD 文档。仅需明确：
- 页面主要功能是什么？
- 有哪些关键交互？
- 需要展示哪些数据？

**操作指令**:
> "我要做一个[功能描述]的单页应用，请帮我梳理简要需求。"

---

## 阶段 2：UI/UX 设计

### 2.1 确认设计规范
- 引用 `brand-guidelines` skill（如有）获取颜色和字体规范。
- 确定布局结构（如：顶部导航 + 内容区 + 底部）。

### 2.2 生成视觉参考 (可选)
使用 DALL-E 3 生成视觉参考图（如果需要灵感）。

**操作指令**:
> "请根据需求，结合我的品牌规范，设计一个页面布局方案。"

---

## 阶段 3：代码实现

### 3.1 创建單文件 HTML
直接创建一个包含 HTML、CSS (Tailwind CDN)、JS 的单文件。

**技术栈**:
- HTML5
- TailwindCSS (通过 CDN 引入，无需构建)
- Vue 3 / Alpine.js (通过 CDN引入，适合轻量级交互)

### 3.2 编写代码
1. 创建 `index.html`。
2. 引入 Tailwind CDN: `<script src="https://cdn.tailwindcss.com"></script>`
3. 编写结构与样式。
4. 添加交互逻辑。

**操作指令**:
> "请帮我生成 index.html，使用 TailwindCSS 和 Vue 3 (CDN)，实现上述设计。"

---

## 阶段 4：预览与交付

1. 直接在浏览器打开 `index.html` 进行预览。
2. 调整细节直至满意。

---

**适用场景**:
- 演示 Demo
- 营销落地页 (Landing Page)
- 简单的计算器/工具
- 个人简历页面
