# PRD 编写规范

## ✅ 编写 PRD 的核心标准（Checklist）

1. **结构完整度**：不允许省略上述模板的任何一个主要章节。
2. **逻辑边界无疏漏**：在“**5. 页面详细逻辑**”部分的表格中，必须明确描述每一种可能的**异常动作**和**边界情况**（例如，网络错误、字段为空、输入过长、无权限等），切忌只写“正向成功流”。
3. **消除冗余，极致复用**：对于全站通用的加载遮罩、二次确认弹窗、错误提示样式等，务必上提至“**4. 全局规范**”中进行阐述，不可在具体页面模块里反复说明。
4. **验收用例的可执行性**：在“**8. 验收标准**”中，不能写诸如“满足用户需求”的空话。每一条都应该是可以交给测试人员当做 Test Case 执行的特定业务场景描述。
